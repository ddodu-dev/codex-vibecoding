# 오늘의 코딩 퀘스트 작업 지침

## 프로젝트 목표

이 프로젝트는 React로 만드는 작은 “오늘의 코딩 퀘스트” 앱입니다.

사용자는 버튼을 눌러 오늘의 코딩 퀘스트 카드를 랜덤으로 뽑고, 카드의 등장·퇴장 애니메이션과 버튼 눌림 효과를 확인하며, 현재 퀘스트의 완료 상태를 변경할 수 있어야 합니다. 1차 구현은 이 핵심 흐름이 작고 명확하게 동작하는 것을 목표로 합니다.

## 작업 원칙

- 구현 전에 `package.json`, `src/App.jsx`, `src/index.css`가 존재하는지 확인합니다.
- 현재 앱 스캐폴드가 없다면 임의로 구현을 시작하지 말고, 필요한 복원 또는 생성 범위를 먼저 확인합니다.
- 변경은 `src/App.jsx`와 `src/index.css`를 중심으로 작고 집중되게 유지합니다.
- Motion for React의 현재 공식 문서를 기준으로 구현합니다.
- 첫 구현은 단순하게 유지하되 실제로 동작해야 합니다.
- 작업 후 변경 파일, 참고한 문서 내용, 브라우저 확인 방법과 남은 위험을 요약합니다.

## 1차 기능

- 오늘의 코딩 퀘스트 카드를 표시합니다.
- 미리 정의된 퀘스트 중 하나를 랜덤으로 뽑습니다.
- 카드가 나타날 때 등장 애니메이션을 적용합니다.
- 카드가 교체되거나 사라질 때 퇴장 애니메이션을 적용합니다.
- 버튼을 누르는 동안 시각적인 눌림 효과를 적용합니다.
- 현재 퀘스트의 완료·미완료 상태를 표시하고 변경할 수 있게 합니다.

## Motion for React 규칙

- 추가할 수 있는 애니메이션 라이브러리는 `motion`뿐입니다.
- 설치 명령은 `npm install motion`입니다.
- React용 import 경로는 `motion/react`입니다.
- 기본 import는 다음 형식을 사용합니다.

  ```jsx
  import { motion, AnimatePresence } from "motion/react"
  ```

- 일반 HTML 요소는 `motion.div`, `motion.button`과 같은 motion component로 바꿔 사용합니다.
- `initial`은 요소가 처음 나타나기 전의 시작 상태를 정의합니다.
- `animate`는 화면에 표시될 때 도착할 상태를 정의합니다.
- `exit`는 요소가 React 트리에서 제거될 때의 퇴장 상태를 정의합니다.
- `whileTap`은 버튼을 누르는 동안의 상호작용 효과를 정의합니다.
- 퇴장 애니메이션이 필요한 조건부 렌더링은 `AnimatePresence` 내부에 둡니다.
- 교체되는 카드에는 퀘스트를 식별하는 안정적이고 고유한 `key`를 부여합니다.

## 제외 범위

- `framer-motion`을 설치하거나 import하지 않습니다.
- `motion` 외의 새로운 라이브러리를 추가하지 않습니다.
- 서버, API 또는 데이터베이스를 연동하지 않습니다.
- 로그인, 영구 저장, 공유, 점수 또는 연속 달성 기록을 추가하지 않습니다.
- 퀘스트 작성·편집, 검색, 필터 또는 카테고리 관리 기능을 추가하지 않습니다.
- 드래그 및 스크롤 기반 애니메이션을 구현하지 않습니다.
- SVG 애니메이션을 구현하지 않습니다.
- 복잡한 variants 구조, 수동 애니메이션 제어 또는 레이아웃 애니메이션을 1차 구현에 포함하지 않습니다.
- 작은 요청을 처리하기 위해 프로젝트 전체를 다시 작성하지 않습니다.

## 검증

- 앱이 오류 없이 실행되고 프로덕션 빌드가 성공하는지 확인합니다.
- 랜덤 뽑기 버튼을 누르면 카드 내용이 변경되는지 확인합니다.
- 새 카드가 `initial`에서 `animate` 상태로 자연스럽게 등장하는지 확인합니다.
- 카드 교체 시 `AnimatePresence`와 `exit`가 퇴장 애니메이션을 실행하는지 확인합니다.
- 버튼을 누르는 동안 `whileTap` 효과가 보이는지 확인합니다.
- 완료 상태 변경이 화면에 즉시 반영되는지 확인합니다.
- 빠르게 연속으로 조작해도 카드 전환과 완료 상태가 깨지지 않는지 확인합니다.
- `framer-motion`이 설치되거나 import되지 않았는지 확인합니다.
- 콘솔에 런타임 오류나 React `key` 경고가 없는지 확인합니다.

## 예상 위험

- 앱 스캐폴드가 없는 상태에서는 소스 파일과 패키지 구성을 먼저 마련해야 합니다.
- `AnimatePresence`의 위치가 잘못되면 퇴장 애니메이션이 실행되지 않을 수 있습니다.
- 카드의 `key`가 안정적이지 않으면 전환 애니메이션이나 상태가 예상과 다르게 동작할 수 있습니다.
- 빠른 연속 클릭으로 애니메이션이 겹칠 수 있습니다.
- 순수 랜덤 선택은 같은 퀘스트를 연속으로 뽑을 수 있습니다.
- 과도한 이동이나 확대 효과는 사용성과 접근성을 떨어뜨릴 수 있으므로 애니메이션을 짧고 절제되게 유지합니다.
